Back-end Terminologies

R - RESTful API

An architectural style for designing networked applications. It relies on a stateless, client-server, cacheable communications protocol -- and in virtually all cases, the HTTP protocol is used.

R - Repository

A central location in which data is stored and managed. Often used to store and manage code in version control systems like Git.

F - Framework

A platform for developing software applications. It provides a foundation on which software developers can build programs for a specific platform.

F - Function

A block of code that performs a specific task. Functions are reusable and can be called multiple times within a program.

N - Node.js

A JavaScript runtime built on Chrome's V8 JavaScript engine. It allows developers to use JavaScript to write server-side code.

N - NPM (Node Package Manager)

A package manager for JavaScript that allows developers to share and reuse code, as well as manage project dependencies.

B - Backend

The server-side of an application. It includes the server, database, and application that communicate with the front-end.

B - Bug

An error, flaw, or fault in a computer program that causes it to produce an incorrect or unexpected result.

V - Virtual Machine

A software emulation of a physical computer. It runs an operating system and applications just like a physical computer.

V - Version Control

A system that records changes to a file or set of files over time so that you can recall specific versions later.
